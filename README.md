# HP设计院屎山数据库整理工具 V1.0 (HP-Design Shit-Mountain Database Sorter V1.0)

![Python Version](https://img.shields.io/badge/Python-3.8%2B-blue)
![License](https://img.shields.io/badge/License-MIT-green)
![LLM](https://img.shields.io/badge/AI_Engine-Kimi_(Moonshot)-orange)

本软件是一款专为**建筑/景观设计院**打造的本地历史数据智能化整理工具。
面对堆积如山、命名杂乱、层级极深的旧项目文件，本引擎利用大语言模型（LLM）的语义理解能力，配合多维本地元数据探测技术，自动将非标准文件“搬运并归类”至设计院标准的数据节点库中，同时确保项目源文件的绝对安全。

---

## ✨ 核心特性 (Core Features)

1. 🧠 **AI 语义锚定与打捞**：告别传统的死板正则匹配，利用大模型理解上下文与文件命名的真实意图。
2. 🛡️ **InDesign 防断链机制**：自动锁定包含 `Links` 和 `Document fonts` 的排版资产包，保持整体搬运，不破坏打包关联。
3. ⏱️ **多维时间嗅探器**：综合正则表达式、照片 EXIF、PDF 内部元数据及系统时间，提取最真实的文件产出时间。
4. 🏥 **强悍的智能自愈与熔断层**：内置细粒度质检（QA Checkers），自动识别 API 限流、大模型“幻觉”（空返回/乱编格式），触发自动重跑与熔断，支持夜间无人值守挂机处理。
5. 🚀 **前后端分离架构与并发池**：现代化 `CustomTkinter` UI 界面绝不卡死，后台 `ThreadPoolExecutor` 动态并发拉满。
6. ✨ **极度强迫症友好**：阶段三内置零散文件精准回收，以及“幽灵空壳文件夹”一键递归粉碎功能。

## 🏗️ 架构与思路解析 (Architecture & Logic)

本引擎严格遵循 **“隔离协议”**：前端只负责数据流转与日志展示，后端 `core_engine` 只接受传参并闷头干活。
业务流转被拆分为 **三大阶段（UI 呈现）** 与 **五大步骤（底层执行）**：

### 📍 阶段 1：算力中心 (生成锚定表)
* **Step 01 - 多维扫描 (`step01_scanner.py`)**：快速遍历源项目，提取文件骨架树，生成带时间戳和体积的字典，极大压缩给 LLM 喂送的数据量（Token）。
* **Step 02 - AI 锚定 (`step02_ai_anchor.py`)**：去除纯数字层级去噪后，分批次发送给 Kimi API 进行归类判定，生成 `[项目名]_锚定表.json`。
  * *内置自适应补丁：自动识别 Kimi 推理模型并锁定温度为 1.0 以防报错。*

### 📍 阶段 2：集中汇流 (物理复制)
* **Step 03 - 物理拷贝 (`step03_physical_copy.py`)**：根据人工复核后的 JSON 锚定表，从源文件夹以 `shutil.copy2` 进行**纯复制**（防损坏旧数据），汇流到新 NAS 标准目录。具备同名文件 `_副本X` 自动防覆盖机制。

### 📍 阶段 3：终极扫尾 (打捞与粉碎)
* **Step 04 - AI 打捞 (`step04_salvage_ai.py`)**：针对新 NAS 中被遗弃在 `06其他` 里的散落文件，利用后缀强字典和语义法则进行二次深度扫描与鉴定。
* **Step 05 - 物理移动 (`step05_salvage_move.py`)**：将有价值的资产从垃圾桶移动到目标大类，支持 UUID 防重名，并在最后彻底删除产生的所有空文件夹。

## ⚙️ 环境配置与安装 (Installation)

本项目推荐使用 **uv** 进行极速依赖管理与本地虚拟环境配置：

1. **安装 Python 3.8+**，并确保将其添加至系统环境变量。
2. 克隆或下载本代码仓库，并在根目录打开终端。
3. **创建并激活虚拟环境**：

```bash
uv venv
# Windows 激活虚拟环境:
.venv\Scripts\activate
# macOS/Linux 激活虚拟环境:
source .venv/bin/activate
```

4. **安装核心依赖**：
```bash
uv pip install -r requirements.txt
```

5. **配置 API 密钥**：
初次运行前或在软件界面中，你需要准备一个月之暗面 (Moonshot) 的 API Key。

## 📖 详细使用指南 (User Guide)

通过运行主程序唤起界面：
```bash
python main.py
```

### 📌 Tab 1: 阶段 1 (生成文件夹锚定表)
**目标：仅仅消耗算力，提取项目的归类映射地图。**

1. 在左侧文件树中找到需要整理的【旧项目文件夹】（可多选）。
2. 点击“➡️ 添加”将其列入待处理清单。
3. 在上方填入**该批次项目的操作人/作者姓名**（如：`张三`）。
4. 检查右上方 API 设置（并发数默认 3，可根据网络情况适当调整）。
5. 点击 **“🚀 启动解析”**。
6. **人工复核（极其重要）**：系统跑完后，会在各源项目旁边生成 `XXX_锚定表.json`。请使用代码编辑器（如 VSCode）打开这些 JSON 文件，人工检查 AI 归类的结果是否符合预期，若有误判请手动修正后保存。

### 📌 Tab 2: 阶段 2 (复制到新文件夹)
**目标：依据复核后的 JSON，将文件安全无痛地拷贝到新阵列中。**

1. 设定最上方的 **“全局新文件夹根目录”**（例如：`D:\新NAS库`）。
2. 点击“➕ 批量选择 JSON”，选中刚才复核完毕的锚定表。系统会自动尝试寻找它们对应的旧源文件夹路径（也可点击“从阶段 1 导入”）。
3. （可选）如果你想让某个项目放到特定的位置，可以单独点击“修改”调整其独立目标路径。
4. 点击 **“🚀 执行复制”**。程序会开始纯拷贝作业，过程随时可见日志进度。

### 📌 Tab 3: 阶段 3 (整理零散文件)
**目标：榨干被遗弃文件的价值，并让文件夹保持强迫症级别的干净。**

1. 点击“⬇️ 从阶段 2 导入”，直接把刚才拷贝好的【新文件夹项目路径】引入列表；或者手动从左侧目录树添加。
2. 点击 **“🧹 零散文件识别和整理”**。
3. 系统会自动穿透到该项目的 `06其他` 中寻找落单文件，重新执行 AI 判断，并在成功后执行**物理剪切移动**。
4. 最终会自动清理新项目中所有遗留的空文件夹。

## ❓ 常见问题与排错 (FAQ)

**Q1: Kimi API 频繁报错 `401 Invalid Authentication`**
* **原因**：你的 `config.ini` 或界面中填写的 `KIMI_API_KEY` 不正确、复制时带了空格，或者账户已被封禁/欠费。
* **解决**：前往月之暗面开放平台重新生成 Key 并在软件中替换保存。

**Q2: Kimi API 报错 `400 invalid temperature: only 1 is allowed...`**
* **原因**：官方近期更新了限制，`kimi-k` 开头的模型强制要求 `temperature=1.0`。

**Q3: 为什么有些文件直接被略过没有拷贝？**
* **原因**：程序默认忽略了以下垃圾格式：`.bak`, `.dwl`, `.dwl2`, `.tmp`, `.err`, `.log`, `.recover`，以及以 `~$` 开头的缓存文件，这属于主动净化功能。

**Q4: 程序运行一半报错卡住了怎么办？**
* **容错机制**：系统内置了 `max_retries`。某一批次偶尔超时，会自动重试 3 次；如果彻底崩溃，外层的文件夹级防线会将其捕获，重新处理整个项目 2 次。只有真的彻底断网或余额耗尽，才会中止**当前项目**并继续跑下一个。

**Q5: GUI 界面点击运行后没有反应？**
* **解决**：由于采用多线程设计，主界面的响应不会卡顿。请密切关注界面底部的黑色日志框，所有的真实处理进度都会实时打印在其中。
